# 宝宝的一万天 ✨

一个为老婆的第 10000 天（1999.03.03 → 2026.07.19）准备的浪漫动态网页。
可爱俏皮风，支持鼠标操作与手机浏览，可用 GitHub Pages 免费发布。

---

## 一、本地预览（电脑上用鼠标模拟操作）

直接双击 `index.html` 一般也能看，但**详情页照片加载**需要本地服务器。推荐任选一种：

**方式 A（有 Python，Mac 自带）：**
在本文件夹打开终端，执行：
```bash
cd "此文件夹路径"
python3 -m http.server 8080
```
然后浏览器打开 http://localhost:8080

**方式 B（有 VS Code）：** 安装 “Live Server” 插件，右键 `index.html` → Open with Live Server。

### 操作方式
- 点击「开启一万天之旅」进入
- ▶ 自动巡航（镜头自动一站站往右走）；也可 ⏸ 暂停
- ⟪ ⟫ 上一站 / 下一站
- **鼠标拖动画面**、**滚轮**、**键盘方向键** 都能移动镜头
- 点节点上的图标或卡片 → 进入该节点的故事详情页（浏览器返回键可退回）
- 走到第 10000 天会出现横幅、海底捞火锅和庆祝彩带 🎉

## 二、发布到 GitHub Pages（手机上查看，无需公网 IP）

1. 在 GitHub 新建一个仓库，例如 `ten-thousand-days`（Public）
2. 把本文件夹里的所有文件上传上去（保持目录结构：`index.html` 在根目录）
   - 网页版：仓库页面 → Add file → Upload files → 拖入全部文件 → Commit
   - 或用命令行：
     ```bash
     git init
     git add .
     git commit -m "宝宝的一万天"
     git branch -M main
     git remote add origin https://github.com/你的用户名/ten-thousand-days.git
     git push -u origin main
     ```
3. 仓库 → Settings → Pages → Source 选 `Deploy from a branch` → 分支选 `main`、目录选 `/ (root)` → Save
4. 等 1~2 分钟，页面会给出网址，形如：
   `https://你的用户名.github.io/ten-thousand-days/`
5. 把这个网址发到手机（微信/短信皆可），手机浏览器打开即可，已做手机适配 📱

> 提示：仓库必须 Public，Pages 才免费可用。

## 三、如何补充内容（都很简单）

所有文字、图片配置集中在 `js/data.js`：

- **改文字**：修改每个节点的 `story` 字段（`\n` 表示换行，`{{day}}` 会自动替换成天数）
- **主线 / 支线切换**：改节点的 `type` 字段
  - `'main'`   = 主线：大图标 + 详细故事页
  - `'branch'` = 支线：小图标、低调、不抢主线视线，但仍显示天数和百分比
  - `'future'` = 未来憧憬：梦想节点
- **有图 / 无图**：加 `noPhoto:true` 表示不留图位；去掉它就会有图位（可上传）
- **我方预置照片**：把图片放进 `photos/`（文件名即日期，如 `20140502.jpg`），把该节点 `photo:''` 改成 `photo:'文件名.jpg'`（所有人都能看到）
- **她自己上传的照片**：进入某节点详情页点「📤 上传照片」即可，保存在她自己手机浏览器里（见下方说明）
- **加背景音乐**：一首 mp3 命名 `bgm.mp3` 放到 `assets/`，点面板 🔈 播放

### 关于「图片上传」的重要说明
- 上传的照片保存在**当前这台设备的浏览器本地**（localStorage），换设备/清缓存会消失。
- 所以：**想让所有人都能看到的照片**（比如你精选的），请用「我方预置照片」方式放进 `photos/` 并提交到 GitHub；
  **她想临时自己加的照片**，用页面里的「上传照片」按钮即可，存在她自己手机上。
